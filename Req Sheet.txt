# 🗃️ DATABASE (POSTGRESQL)

## Users

```sql
id SERIAL PRIMARY KEY
name
email UNIQUE
password
created_at
```

## Restaurants

```sql
id SERIAL PRIMARY KEY
name
rating
delivery_time
created_at
```

## Menu_Items

```sql
id SERIAL PRIMARY KEY
restaurant_id REFERENCES restaurants(id)
name
price
description
```

## Orders

```sql
id SERIAL PRIMARY KEY
user_id REFERENCES users(id)
status
total_amount
eta
timeout_at
created_at
updated_at
```

## Order_Items

```sql
id SERIAL PRIMARY KEY
order_id REFERENCES orders(id)
menu_item_id REFERENCES menu_items(id)
quantity
price
```

* Use environment variables (.env)
* Validate all inputs (e.g. Joi/Zod)
* Use proper HTTP status codes
* Centralized error handling middleware
* Use async/await with proper error handling
* Prevent race conditions in order updates
* Use transactions for order creation

---

## Database (PostgreSQL)

* Use normalized schema (no redundancy)
* Use indexes on:

  * foreign keys
  * frequently queried fields
* Use constraints:

  * NOT NULL
  * UNIQUE
* Maintain referential integrity
* Use transactions where required

---

## Frontend

* Use clean component structure
* Separate UI and logic
* Use reusable components
* Avoid unnecessary re-renders
* Handle loading and error states
* Use Axios interceptors for auth

* Avoid storing derived data
* Use async thunks properly

* Follow REST conventions

* Do not expose sensitive data
